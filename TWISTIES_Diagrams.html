3. selectWPs() Algorithm Flow (Option C)

selectWPs(roads, baseRoute, curv, maxDur) │ ├─ 1. FILTER CANDIDATES │ ├─ maxC = 15 + (curv/100)*45 │ ├─ Keep: distance_to_route <= maxC │ └─ Keep: scorePerKm > 0 │ └─► Reduces 500 roads → ~50-100 candidates │ ├─ 2. DISTRIBUTE INTO ZONES │ ├─ Create 8 empty zones │ └─ For each candidate: │ └─ Assign to zone[floor(progress*8)] │ └─► Spreads candidates evenly along route │ ├─ 3. RANK WITHIN ZONES │ └─ For each zone: │ ├─ Sort by scorePerKm (curviness) │ ├─ Use distance as tiebreaker │ └─ Take top 1-2 per zone │ └─► Reduces 50-100 → ~20 top candidates │ ├─ 4. PRE-FILTER │ └─ Sort all top candidates by scorePerKm │ └─ Keep only top 20 │ └─► Final candidates list ready for testing │ ├─ 5. PARALLEL BATCH TESTING │ ├─ Batch size = 4 │ └─ For each batch: │ ├─ Create Promise for each candidate │ │ ├─ Current selection + this candidate │ │ └─ Call OSRM(selection + candidate) │ ├─ Wait for all 4 in parallel │ │ └─ Promise.all([osrm(c1), osrm(c2), osrm(c3), osrm(c4)]) │ └─ Keep candidates where dur <= maxDur │ └─► OPTIMIZATION #2: Parallel requests instead of sequential │ └─ 6. RETURN └─ Selected candidates sorted by progress

5. Time Budget Constraint

User sets sliders: ├─ Curviness: 50% └─ Detour Budget: +30% These map to time constraints: ├─ Baseline route (direct): 340 minutes └─ Max allowed with detour: 340 × (1 + 30/100) = 442 minutes Time Available ┌──────────────────────────────┐ │ │ 0 ┌─┴──────────────────────────────┴─┐ │ │ │ Baseline (direct) │ │ 340 min │ │ ─────────────────────────────── │ │ │ │ Twisty route with waypoints │ │ 390 min (still within budget) │ │ ──────────────────────────────── │ │ │ │ BUDGET BUFFER │ │ (442 - 390) = 52 min remaining │ │ ──────── │ │ │ 442 └──────────────────────────────────────┘ Max allowed (baseline × 1.30) Waypoints only selected if: final_route_time <= baseline_time × (1 + detour_pct/100)

6. Curvature Scoring Algorithm (Circumradius)

For each road, iterate through 3-point segments: P1 / \ / \ / \ ← circumradius (R) / \ / \ ────P2──────────P3──── Sharp turn (small R): Gentle curve (large R): ╱╲ ╱╲ ╱ ╲ ╱ ╲ ╱ ╲ R = 0.02 km ╱ ╲ R = 0.5 km ╱ ╲ ╱ ╲ ───────── ───────── weight = 4 weight = 1 Weight buckets (by radius): ┌─────────────┬───────────────┐ │ Radius │ Weight │ ├─────────────┼───────────────┤ │ < 0.03 km │ 4 (very hot) │ │ < 0.06 km │ 3 (hot) │ │ < 0.175 km │ 2 (medium) │ │ < 0.5 km │ 1 (mild) │ └─────────────┴───────────────┘ Final score = Σ(segment_length × weight) for each road Score per km = total_score / total_road_length Higher score per km = more twisty
